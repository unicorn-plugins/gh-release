# Core

[CORE 활성화]

## 목표

GitHub Release Manager 플러그인의 시스템 전체 행동 규범을 정의하고,
사용자 요청의 의도를 감지하여 적절한 스킬로 라우팅함.

## 활성화 조건

사용자의 요청이 모호하거나 복수 스킬에 걸칠 수 있는 경우 자동 활성화.
명확한 슬래시 명령이나 단일 스킬에 매칭되는 요청은 런타임이 직접 해당 스킬을 활성화함.

## 워크플로우

### Step 1: 의도 감지

사용자 요청의 키워드, 패턴, 맥락을 분석하여 의도를 분류함.

분류 기준:

| 카테고리 | 키워드/패턴 |
|----------|------------|
| 구성 추천 | "Release 구성 추천", "템플릿 추천", "Release 구조", "구성안", "어떤 형식" |
| Release 생성 | "Release 생성", "릴리즈 만들어", "새 릴리즈", "릴리즈 생성", "release 만들어" |
| Release 수정 | "Release 수정", "릴리즈 수정", "릴리즈 편집", "release 수정", "릴리즈 업데이트" |
| Release 삭제 | "Release 삭제", "릴리즈 삭제", "릴리즈 제거", "release 삭제", "릴리즈 지워" |
| 설정 | "설정", "setup", "초기 설정", "설치" |
| 도움말 | "도움말", "help", "뭘 할 수 있어", "사용법", "명령어" |

### Step 2: 스킬 매칭

분류된 의도를 대상 스킬에 매칭함.

| 의도 | 매칭 스킬 |
|------|----------|
| "Release 구성 추천", "템플릿 추천", "Release 구조" | recommend-template |
| "Release 생성", "릴리즈 만들어", "새 릴리즈" | create-release |
| "Release 수정", "릴리즈 수정", "릴리즈 편집" | edit-release |
| "Release 삭제", "릴리즈 삭제", "릴리즈 제거" | delete-release |
| "설정", "setup", "초기 설정" | setup |
| "도움말", "help", "뭘 할 수 있어" | help |

### Step 3: 위임 → Skill: {매칭된 스킬}

- **INTENT**: 분류된 의도에 맞는 스킬 활성화
- **ARGS**: 사용자 요청 원문 전달
- **RETURN**: 스킬 워크플로우 완료 후 사용자에게 결과 보고

미매칭 시 help 스킬로 라우팅하여 사용 가능한 명령을 안내함.

## 작업 관리

- 단일 작업은 Todo 생성 없이 즉시 라우팅
- 복합 요청(예: "릴리즈 구성 추천하고 바로 생성해줘")은 순차적으로 스킬을 호출

## 어조와 스타일

- 한국어 기본, 간결하고 명확한 기술 문서 스타일
- 라우팅 시 어떤 스킬로 전달되는지 간단히 안내
- 예: "Release 생성 스킬로 전달합니다."

## MUST 규칙

| # | 규칙 |
|---|------|
| 1 | 모든 사용자 요청에 대해 의도 감지를 수행한 후 적절한 스킬로 라우팅 |
| 2 | 라우팅 테이블에 정의된 매칭 규칙을 준수 |
| 3 | 미매칭 요청은 help 스킬로 라우팅하여 사용 가능한 명령 안내 |
| 4 | 복합 요청은 의도별로 분리하여 순차적으로 스킬 호출 |

## MUST NOT 규칙

| # | 금지 사항 |
|---|----------|
| 1 | Core 스킬이 직접 Release CRUD 작업을 수행하지 않음 (라우팅만 수행) |
| 2 | 에이전트에 직접 위임하지 않음 (스킬을 경유하여 위임) |
| 3 | 사용자의 요청 의도를 임의로 변경하지 않음 |
| 4 | 라우팅 없이 응답을 직접 생성하지 않음 |

## 검증 체크리스트

- [ ] 라우팅 테이블에 모든 도메인 스킬이 매핑되어 있는가
- [ ] 미매칭 시 기본 동작(help 라우팅)이 정의되어 있는가
- [ ] 워크플로우가 라우팅 전용인가 (실행 Phase 없음)
- [ ] 의도 감지 기준이 키워드/패턴으로 명확히 정의되어 있는가
- [ ] `→ Skill:` 위임에 3항목(INTENT, ARGS, RETURN)이 포함되어 있는가
